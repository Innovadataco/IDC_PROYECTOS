## 8. METODOLOGÍA: PM2 (PROJECT MANAGEMENT v2)

Cada proyecto en InnovaDataCO sigue una estructura replicable:

1. **INICIAR**: Acta inicial, alcance, equipo.
2. **PLANIFICAR**: Plan de trabajo, cronograma, entregables.
3. **EJECUTAR**: Desarrollo, seguimiento, control.
4. **VALIDAR**: Revisión, ajustes, aprobación.
5. **CERRAR**: Acta de cierre, lecciones aprendidas.

### 8.1 Candados de Gobierno

1. ZEUS no declara "completado". Solo declara "listo para validación".
2. ODIN valida con Acta. Sin `ACTA-VALIDACION-XXX.md`, no hay merge ni siguiente módulo.
3. Jelkin firma instrucción. Sin `INSTRUCCION-XXX.md` firmada, ZEUS no empieza módulo nuevo.

## 9. FORMAS DE OPERAR

### 9.1 Protocolo de Sincronización

> **GitHub = Verdad Absoluta**

Antes de CUALQUIER acción de proyecto:
1. `git pull origin main`
2. Verificar estado local = remoto
3. Recién entonces: trabajar, editar, actualizar
4. Inmediatamente después: `git add + commit + push`

### 9.2 Ritual Diario Obligatorio (ZEUS)

1. Sincronizar repo: `git pull` en `IDC_PROYECTOS`.
2. Preguntar al CEO: *"¿Avances hoy? ¿Qué proyectos avanzaron?"*
3. Actualizar PM2 con avances.
4. Pushear inmediatamente.
